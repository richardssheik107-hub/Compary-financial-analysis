# Step 13 说明文档：提交后二次重跑稳定性修复

## 1. 目标

修复两个体验问题：

- 顶部上下文栏在提交后未实时更新
- 延申问题/结果区偶发前端渲染异常（DOM removeChild 错误）

## 2. 方案

采用 Streamlit 表单推荐模式：

1. 提交时先执行分析并写入 `session_state`
2. 立即调用 `st.rerun()`
3. 下一轮统一渲染上下文栏与结果区

## 3. 代码改动

- `app.py`
  - 提交分支不再“当轮渲染结果”
  - 改为“写状态 -> rerun -> 次轮渲染”
  - 新增 `last_render_concise_only` 记录渲染模式，保证次轮显示一致

## 4. 效果

- 上下文栏在每次提交后可实时看到最新公司/对比对
- 结果区渲染路径统一，降低前端树抖动概率

## 5. 验收标准

- 提交后上下文栏实时更新
- 延申问题展示稳定
- 冒烟检查通过
