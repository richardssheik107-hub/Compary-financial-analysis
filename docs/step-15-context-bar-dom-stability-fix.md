# Step 15 说明文档：顶部上下文栏 DOM 稳定性修复

## 1. 问题

- 页面出现前端 DOM 异常（removeChild）
- 顶部上下文栏在异常状态下不稳定

## 2. 修复策略

- 将上下文栏从自定义 HTML 卡片渲染改为 Streamlit 原生组件渲染（`st.write` + `st.button`）。
- 保留原有信息内容，不再依赖复杂 HTML 节点树。

## 3. 结果

- 减少重跑场景下前端节点重排风险
- 上下文栏展示路径更稳定、可维护性更高

## 4. 验收标准

- 应用可正常加载
- `/_stcore/health` 返回 `ok`
- 顶部上下文栏可持续显示
