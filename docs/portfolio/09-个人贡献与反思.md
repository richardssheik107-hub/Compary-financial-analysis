# 09 个人贡献与反思

## 我的角色

- 角色定位：产品设计 + 技术实现 + 迭代推进（端到端 owner）。  
- 时间投入：从需求定义、功能实现、问题修复到评测文档全链路参与。

## 个人贡献

1. 产品侧  
- 定义三类核心场景（单公司/对比/通用）和对应体验目标。  
- 设计对比页核心信息结构（双K线、对比表、结论卡、未来看法）。  
- 补齐演示脚本、面试话术和简历表述。

2. 技术侧  
- 实现 Agent 路由主链路与上下文记忆机制。  
- 完成对比能力从“识别”到“完整渲染”的升级。  
- 实现失败兜底与稳定输出策略。

3. 工程管理侧  
- 建立 Step 文档与工作日志机制。  
- 建立评测基线（测试集 + 脚本 + 指标），让优化可量化。  
- 多轮版本回退与恢复，保证项目可持续推进。

## 最有价值的决策

- 决策：把项目从“单次输出”升级为“场景化 Agent 产品”。  
- 价值：让系统具备可解释链路、可复用流程和可量化迭代空间。  
- 结果：对比链路稳定，具备了面试可展示的产品与工程闭环。

## 反思

1. 做得好的地方  
- 能快速把用户反馈转成具体迭代动作并落地。  
- 文档、日志、代码同步推进，项目可复盘性强。

2. 可以更好的地方  
- 文本质量策略应该更早引入评测门控。  
- 单公司意图识别规则还需进一步提高泛化能力。

3. 如果再做一轮，我会优先做什么  
- 先建评测体系再做大规模文本优化。  
- 增加用户测试样本，减少“开发者视角偏差”。

## 下一步路线图

- 1个月内：  
  - 提升意图准确率到 85%+  
  - 提升文本质量通过率到 80%+  
  - 补齐演示视频和案例截图

- 3个月内：  
  - 建立高质量语料与改写质量门控  
  - 引入更强事实一致性评估  
  - 支持更多行业和多轮复杂追问
