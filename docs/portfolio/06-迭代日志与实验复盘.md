# 06 迭代日志与实验复盘

## 关键迭代记录

### 迭代A：Agent 基线搭建
- 背景：初始版本只有基础查询能力，缺少可解释的执行流程。  
- 假设：加入统一 Agent 入口与步骤追踪能提升可维护性与可讲述性。  
- 改动：引入 `run_financial_agent`、`AgentResult`、执行步骤展示。  
- 结果：形成可观察链路，为后续对比与追问能力奠定基础。  
- 结论：结构化编排优于“页面直接拼逻辑”。

### 迭代B：双公司对比能力增强
- 背景：用户提出“能识别对比，但内容没有真正对比”。  
- 假设：补齐双K线、对比财务表、对比点评后，结果可解释性显著提升。  
- 改动：新增对比页专用渲染分支与对比结论卡。  
- 结果：对比成功率达到 100%（基线评测）。  
- 结论：对比场景必须提供“证据 + 结论”组合。

### 迭代C：上下文与稳定性修复
- 背景：追问中“它/这家公司”无法正确继承，且出现前端渲染不稳定。  
- 假设：标准化上下文状态并使用更稳妥渲染路径可降低故障。  
- 改动：统一 `ConversationContext`；调整前端渲染策略与回退逻辑。  
- 结果：追问链路恢复，页面可用性提升。  
- 结论：会话状态是 Agent 产品体验核心，不可后置。

## 一次失败实验（保留）

- 目标：通过快速改动文本生成链路显著提升文案质量。  
- 为什么失败：局部改动叠加编码污染，造成语句异常和不稳定。  
- 学到什么：文本链路要优先“数据与编码卫生”，否则模型优化会被底层问题抵消。  
- 后续如何调整：先建立评测基线和质量门控，再做生成策略迭代。
