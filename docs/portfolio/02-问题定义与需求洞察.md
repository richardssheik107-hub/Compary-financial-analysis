# 02 问题定义与需求洞察

## 用户痛点

1. 财报信息“看得见但看不懂”：术语多、口径复杂，普通用户难以快速得出结论。  
2. 对比效率低：用户需要在多个页面切换，无法在同一视图看到可比信息。  
3. 问法不标准：用户常用口语（如“谁更好”“它怎么样”），系统容易识别失败。  
4. 场景不稳定：未识别公司或数据源异常时，传统问答系统容易直接报错或空输出。

## 场景拆解

### 场景A：单公司分析
- 用户目标：快速判断公司经营质量与未来关注点。  
- 当前难点：财务指标分散，缺少通俗解释与重点提示。  
- 产品策略：输出白话分析、未来看法、评分、雷达图、K线和核心指标表。

### 场景B：双公司对比
- 用户目标：在同口径下判断哪家公司当前更优。  
- 当前难点：对比维度多、信息割裂、结论难统一。  
- 产品策略：输出领先结论、双K线、对比财务表、对比文字点评与未来观察。

### 场景C：通用财务问答
- 用户目标：即使不提公司，也获得可执行的分析方法。  
- 当前难点：传统系统常要求“先给公司名”，体验中断。  
- 产品策略：进入通用模式，输出方法型答案和后续可追问建议。

## 需求优先级

### P0（必须）
- 三路由稳定可用（单公司/对比/通用）。  
- 失败兜底和风险免责声明。  
- 基础可视化与文本输出。

### P1（重要）
- 上下文记忆（追问继承公司）。  
- 对比页信息密度提升（图表+表格+结论）。  
- 评测基线与量化指标。

### P2（可选）
- 文本质量高级优化（改写器/质量门控）。  
- 多轮问答更强记忆。  
- 行业维度扩展与个性化风格。
