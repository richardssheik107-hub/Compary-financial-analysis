# 08 风险与边界

## 金融场景风险

1. 数据时效风险：公开数据更新节奏不一致，可能导致结论滞后。  
2. 口径不一致风险：不同公司/报告期口径差异会影响对比公平性。  
3. 文本幻觉风险：模型可能生成流畅但事实不严谨的描述。  
4. 误用风险：用户可能把解读当作直接投资建议。

## 系统边界

明确不做：
- 直接投资建议  
- 收益承诺  
- 股价预测  
- 自动交易信号输出

## 防护策略

1. 风险提示强制输出：所有分析需包含“不构成投资建议”。  
2. 失败兜底策略：公司未识别或 API 失败时，切换通用模式/本地兜底。  
3. 交易指令词替换：对“买入/卖出/梭哈”等词汇进行替换处理。  
4. 结果可观测：展示 Agent 执行流程，降低黑盒感。  
5. 前端免责声明：页面固定展示风险声明。

## 后续改进

- 增加事实一致性校验（生成文本与原始指标对齐检查）。  
- 增加输出质量门控（低分文本自动重试）。  
- 增加“数据时间戳”展示，提升用户对时效性的认知。
