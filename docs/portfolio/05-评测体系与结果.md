# 05 评测体系与结果

## 评测目标

- 目标1：验证系统在三类场景下的可用性（single / compare / general）。  
- 目标2：量化意图识别和路由稳定性。  
- 目标3：建立文本质量最低标准，支持后续优化对比。

## 指标定义

- 意图准确率（intent accuracy）：预测意图与期望意图一致比例。  
- 路由成功率（route success）：请求得到可用结果（非空、非硬失败）的比例。  
- 对比成功率（compare success）：对比问题中成功生成 `comparison` 的比例。  
- 文本质量通过率（text quality pass）：文本长度、三指标覆盖、风险提示等规则通过比例。  
- 空结果率（empty/error rate）：无可用输出的比例。
