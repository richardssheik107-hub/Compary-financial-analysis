# MVP 验收清单

验收日期：2026-05-17

## 已通过

- Windows 本地虚拟环境 `.venv` 可安装 `requirements.txt`。
- Streamlit 服务可在 `http://localhost:8501` 响应 HTTP 200。
- 页面可手动提交查询，不会打开页面就自动抓取数据。
- 支持 A 股名称和 6 位股票代码识别。
- 支持自然语言中夹带 6 位股票代码，例如“帮我看000001怎么样”。
- 当前 AKShare A 股代码名称表可识别约 5517 家公司。
- 样例查询通过：
  - 贵州茅台：`600519`
  - 平安银行：`000001`
  - 格力电器：`000651`
  - 股票代码：`000001`
- 未知公司会提示“暂未识别到 A 股公司”，不会默认跳回贵州茅台。
- 无 `OPENAI_API_KEY` 时进入本地兜底解读，不崩溃。
- AI 输出评分范围保持在 0-10，前端展示为 0-100。
- 页面显示数据状态、匹配方式、报告期、AI 状态、三项评分、雷达图和风险提示。
- 输出不包含“买入”“卖出”“重仓”“梭哈”等直接交易词。
- `python -m compileall app.py src scripts` 通过。
- 智谱 AI / BigModel `glm-4-flash` 真实 API 冒烟测试通过。
- 浏览器页面已显示真实 API 状态，不再走本地兜底。

## 未执行

- 独立打包 EXE：不属于 MVP 范围。
- 用户登录、交易下单、自选股、回测、实时行情推送：不属于 MVP 范围。

## 残余风险

- AKShare 接口和字段可能随数据源变化，需要持续容错。
- 某些 A 股公司可识别但财务摘要字段不完整，页面会展示可用字段并保留风险提示。
- 部分代码可能属于新股、退市整理、北交所、特殊类型证券或 AKShare 暂未覆盖标的，能识别代码但不一定能获取完整财务摘要。
- Windows 终端可能显示中文乱码，但项目文件本身为 UTF-8，浏览器页面显示正常。
- 真实 API 已完成冒烟测试，但输出质量仍建议继续做多公司人工评审。

## 建议下一步

- 配置 `.env` 中的 `OPENAI_API_KEY`，执行真实 API 场景验收。
- 可优先使用智谱 AI / BigModel 的 `glm-4-flash` 做国内 OpenAI 兼容接口验收。
- 继续完善更多财务字段解释，例如毛利率、资产负债率、研发投入。
- 增加缓存机制，减少 AKShare 重复请求等待时间。
