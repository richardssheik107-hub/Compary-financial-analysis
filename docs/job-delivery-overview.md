# 大白话财报 Agent：投递版项目总览

## 1. 项目一句话

把“财报解读工具”升级为“可解释、可追踪、可追问”的金融分析 Agent，面向普通投资者提供 A 股公司财报理解与双公司对比能力。

## 2. 问题定义与目标用户

- 目标用户：无金融背景或轻金融背景的普通投资者。
- 核心痛点：
  - 财报指标读不懂
  - 不知道该看哪些关键项
  - 对比两家公司时缺少统一口径
- 项目目标：
  - 用自然语言查询公司或对比问题
  - 自动完成识别、取数、解读、评估、追问闭环
  - 明确风险提示，不输出直接交易指令

## 3. 方案演进（Step 1 - Step 6）

### Step 1：Agent 基线

- 新增统一入口 `run_financial_agent(query)`
- 建立最小 SOP：意图识别 -> 数据抓取 -> 财报解读
- 页面新增 Agent 执行过程

### Step 2：工具化与质量评估

- 新增 `ToolRegistry` 统一调用工具
- 增加工具调用明细（成功/失败）
- 增加输出质量评估（风险提示、数据来源、交易词合规、核心指标解释）

### Step 3：双公司对比流程

- 对“对比/比较/谁更好”问题走专用分支
- 自动识别两个公司并输出对比表格与领先结论

### Step 4：可信度与可解释性增强

- 对比结果增加可信度分数、行业语境、限制说明和后续追问建议

### Step 5：交互闭环

- 追问建议升级为“继续问”按钮
- 点击后自动回填并自动触发下一轮分析
- 修复 Streamlit 重跑场景下状态丢失问题

### Step 6：回归保障

- 新增一键冒烟脚本 `scripts/smoke_check.py`
- 覆盖健康检查、单公司、双公司、失败提示四类关键路径

## 4. 当前架构与关键能力

- 前端：`Streamlit + Plotly`
- 数据层：`AKShare + 样例兜底`
- 模型层：`OpenAI 兼容接口（已接入智谱 glm-4-flash）`
- Agent 层：
  - 意图识别
  - 工具编排
  - 质量评估
  - 对比聚合
  - 追问触发

核心能力：

- 单公司白话财报解读
- 双公司核心指标对比
- 输出质量可量化
- 异常和兜底可见
- 追问链路可连续执行

## 5. 验证结果

- 本地服务健康检查通过：`/_stcore/health = ok`
- 单公司链路通过：公司识别、财报解读、图表渲染正常
- 双公司链路通过：对比结论、指标表、可信度与追问建议正常
- 失败场景通过：无法识别两个公司时给出明确提示
- 冒烟脚本通过：4/4 检查全部通过

## 6. 风险边界与后续计划

当前边界：

- 规则意图识别对复杂口语覆盖有限
- 对比结论仍以核心财报指标为主，未纳入完整估值与行业周期模型
- 冒烟检查偏功能通路，不替代完整 UI 自动化测试

后续计划（建议）：

1. 将质量评估规则配置化（便于A/B迭代）
2. 增加公司对比口径校验（报告期一致性强化）
3. 增加请求耗时与工具耗时指标
4. 补一版可投递演示截图和固定 demo 脚本

## 7. 面试讲述提纲（2 分钟）

1. 我做了什么：把财报工具升级成可解释 Agent，支持单公司与双公司对比。
2. 为什么做：用户不只要结论，还要知道系统怎么得出结论，以及下一步怎么问。
3. 怎么做：用 Agent 层串联取数与解读，增加工具调用追踪、质量评估、可信度和追问闭环。
4. 结果如何：核心链路稳定可跑，冒烟脚本 4/4 通过，真实 API 已接入并可演示。
5. 下一步：把评估规则配置化，补充更强的口径校验和可观测性。
