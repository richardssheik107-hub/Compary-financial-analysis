# 大白话财报 Agent：投递版项目总览（最新版）

## 1. 项目一句话
把“财报查询工具”升级为“可解释、可追踪、可迭代”的 AI Agent，支持单公司分析、双公司对比、通用财务问答。

## 2. 目标用户与问题
- 目标用户：非金融专业或轻金融背景用户。
- 核心问题：财报读不懂、指标抓不住、对比没口径、追问没上下文。

## 3. 当前可用能力（最新）
1. 单公司分析：公司识别 -> 数据抓取 -> 大白话解读 -> 风险提示。  
2. 双公司对比：两公司识别、双 K 线、财务指标对比、文字点评与后续问题。  
3. 通用问答：不输入具体公司也可回答；且当前 UI 已改为“只显示答案文本”。  
4. 对比稳态修复：对比识别恢复，支持“万科和格力哪个更好”这类输入。  
5. 别名可配置化：`eval/company_aliases.csv` 可直接维护识别词表。  
6. 运维工具：`alias_check.py` 与 `alias_suggest.py` 支持别名生效检查和失败样本辅助修复。

## 4. 架构与技术栈
- 前端：Streamlit + Plotly  
- 数据：AKShare + 本地兜底数据  
- 模型：OpenAI 兼容接口（当前接入智谱 `glm-4-flash`）  
- Agent：意图识别、工具编排、对比聚合、失败兜底、会话上下文

## 5. 评测与稳定性（当前）
- 评测集：`eval/testset.csv`（20 条）
- 自动评测：`scripts/eval_runner.py`
- 近期指标（当前版本）：
  - intent_accuracy：85%
  - route_success_rate：95%
  - compare_success_rate：83.33%
  - text_quality_pass_rate：95%
  - empty_or_error_rate：5%

## 6. 你可直接演示的亮点
1. 对比问题直接给“图 + 表 + 文字结论”闭环。  
2. 通用问题单独答案展示，交互更干净。  
3. 别名词表可直接改 CSV，不用改代码。  
4. 评测与失败样本可追踪，迭代有数据依据。

## 7. 后续迭代方向
1. 基于失败样本持续扩充别名与意图规则。  
2. 提升 compare 边界样本命中率。  
3. 加强通用问答的事实约束与文本质量门控。  
4. 做一键“新增别名 -> 检查 -> 评测”的流水线脚本。
