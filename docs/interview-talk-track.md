# 面试讲稿（最新版）

## 1 分钟版本
我做的是一个面向 A 股场景的财报分析 Agent。  
它支持三类问题：单公司分析、双公司对比、通用财务问答。  
核心不是“只给结果”，而是把识别、取数、解读做成可追踪流程，并且有失败兜底和风险提示。  
产品上我重点做了对比能力：双 K 线、财务对比表、文字点评一体化输出。  
最近我又把通用问答改成“只显示答案”，并接入 `glm-4-flash` 提升文本质量。  
工程上我建立了自动评测和别名运维工具，当前指标约为 intent 85%、route 95%、text quality 95%，可以持续迭代优化。

## 3 分钟版本
### 1) 为什么做
普通用户看财报最大的障碍不是数据拿不到，而是“看不懂、比不动、追不下去”。  
所以我目标是做一个可解释的财报 Agent，而不是单次问答机器人。

### 2) 做了什么
我把系统拆成三层能力：  
1. 路由层：识别单公司 / 对比 / 通用问题。  
2. 工具层：公司识别、财务抓取、模型解读。  
3. 输出层：图表、对比表、文字结论、风险提示。  

并在双公司场景做了完整产品化：  
- 自动识别两家公司  
- 输出双 K 线  
- 给出财务指标并排对比  
- 附带文字点评和后续追问建议

### 3) 最近关键迭代
1. 修复了双公司对比识别失效问题（编码污染 + 别名映射不稳）。  
2. 将别名词表外置到 `eval/company_aliases.csv`，支持数据化维护。  
3. 新增 `alias_check.py` 与 `alias_suggest.py`，把失败样本修复流程工具化。  
4. 通用问答改为“答案直出”，减少干扰信息。

### 4) 结果
基于 20 条评测集自动回归，当前版本达到：  
- intent_accuracy 85%  
- route_success_rate 95%  
- text_quality_pass_rate 95%

### 5) 下一步
我会继续做两件事：  
1. 用失败样本持续收敛 compare 边界识别。  
2. 增强通用问答的事实约束与答案稳定性。
