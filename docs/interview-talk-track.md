# 面试讲稿

## 1 分钟版本

我做了一个面向 A 股场景的 AI 财报分析 Agent。  
它支持单公司解读、双公司对比和通用财务问答三条链路。  
技术上，我实现了意图识别和工具编排，把公司识别、数据抓取、文本分析组织成可追踪流程，并输出结构化结果。  
产品上，我重点做了对比场景，加入双 K 线图、对比财务表和文字结论，让复杂问题更容易读懂。  
同时我做了上下文记忆和失败兜底，并建立了评测基线来量化质量和稳定性。

## 3 分钟版本

### 1）问题背景
普通用户会问财报问题，但原始报表阅读门槛高，尤其是公司对比时，单条聊天回答不稳定、也不便于决策。

### 2）我做了什么
我把系统拆成 Agent 工作流：
1. 意图路由（单公司 / 对比 / 通用）
2. 工具编排（公司快照 + 文本分析）
3. 结构化结果对象（含执行步骤、告警、建议问题）
4. 按场景渲染（重点是对比页）

### 3）关键产品设计
- 对比场景有专门 UI：领先方原因、双 K 线、财务对比表、对比点评和未来看法。  
- 用户没给公司名时，不报错，自动进入通用模式。  
- 支持上下文记忆，追问时能继承前文目标。

### 4）关键工程设计
- 加了严格兜底策略，保障 API 或数据异常时仍可输出。  
- 建了评测资产（测试集 + 自动评测脚本）做量化迭代。  
- 当前基线：intent 70%、route success 85%、compare success 100%。

### 5）反思与下一步
当前短板主要是单公司意图识别和通用问答文本质量。  
下一步会加强事实约束和文本质量门控，提升整体稳定性。
