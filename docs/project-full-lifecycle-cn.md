# 上市公司分析AGENT：项目全流程文档（中文版）

## 1. 项目定位

本项目是一个面向“公司经营研究”场景的 AI Agent。  
用户输入公司名称、股票代码或自然语言问题，系统自动完成：

1. 意图识别（单公司 / 双公司对比 / 通用经营问答）
2. 数据抓取与兜底（AKShare + 本地样例）
3. 资料增强（RAG 检索本地年报/研报/行业资料）
4. 结构化分析输出（研究报告模板）
5. 可追溯说明（来源、置信度、限制项）

目标不是“给交易建议”，而是把公开信息组织成可复盘的经营研究结论。

## 2. 总体架构

### 2.1 入口层
- `app.py`
  - Streamlit 交互入口
  - 上下文状态管理
  - 可视化展示（执行步骤、K 线、对比表、研究报告）

### 2.2 Agent 编排层
- `src/agent.py`
  - `run_financial_agent(...)`
  - `ToolRegistry`
  - 意图识别、路由、回退策略、对比合成、建议追问

### 2.3 数据层
- `src/data_service.py`
  - 公司识别（代码/别名/模糊）
  - 财务摘要抓取（AKShare）
  - 日线行情抓取（多接口回退）
  - 失败兜底与重试（含代理异常处理）

### 2.4 研究增强层
- `src/research_agent.py`
  - `research_company`
  - `compare_companies_deep`
  - `general_business_qa`
- `src/report_loader.py`
  - 本地资料加载与检索
- `src/research_skill.py`
  - 结构化重写与文本质量优化

### 2.5 模型与数据结构
- `src/models.py`
- `src/research_models.py`
  - `AgentStep`、`AgentResult`
  - `CompanyResearchContext`、`ResearchResult`、`SourceNote`

## 3. 迭代主线（按阶段）

## 阶段 A：Agent 化基线（Step 1~4）
目标：从“能跑”到“可工程化推进”

- Step 1：统一入口 + 执行过程可追踪 + 文档日志机制
- Step 2：工具注册与评测基线
- Step 3：双公司对比流程打通
- Step 4：对比置信度与后续追问建议

阶段结果：形成可路由、可追踪、可回退的最小 Agent 系统。

## 阶段 B：交互与稳定性（Step 5~22）
目标：让功能“可连续使用”

- 继续问交互、追问建议统一
- 会话记忆与上下文栏
- 提交/重渲染稳定性修复
- UI 一致性恢复
- 双公司 K 线与对比展示完善

阶段结果：前端体验可连续对话，可视化完整，错误场景可恢复。

## 阶段 C：质量与可投递化（Step 25~32）
目标：可量化、可复盘、可面试讲述

- 建立评测基线
- 优化意图识别与文本质量
- 别名配置与运维工具化
- 简历投递素材整理

阶段结果：有数据、有文档、有叙事链路，进入“作品级”状态。

## 阶段 D：研究 Agent 升级（Step 33~39）
目标：从财报解读升级为公司经营研究

- 引入研究上下文模型
- 单公司/双公司深度研究路由
- 证据来源与可信度表达
- 保持原 UI 主体，增量加入研究报告

阶段结果：输出从“大白话点评”升级为结构化研究结论。

## 阶段 E：RAG 观测与文本优化（Step 43~57）
目标：提升“有依据”的文本质量

- 增加资料命中观测
- RAG + Skill 消融对比
- 批量扩充本地资料集（公司/行业/报告）
- 输出量化指标与优化报告
- AKShare 稳定性增强（重试、提示）

阶段结果：RAG 增益可量化，系统稳定性与可解释性提升。

## 6. 评测与量化结果（核心）

基于 `N=50` 离线评测集（Base vs Full）：

- `route_success_rate`: 100%
- `text_quality_pass_rate`: 100%
- `avg_total_score`: 84.9 -> 86.3（+1.4）
- `avg_source_count`: 1.3 -> 5.3（+4.0）

说明：RAG + Skill 组合显著增加了来源覆盖和文本信息密度，并带来总分提升。

## 7. 项目中的产品方法

1. 先搭最小闭环，再做能力扩展  
2. 每步有验收条件和回滚策略  
3. 通过评测集验证而非主观判断  
4. 保持“可解释输出”：来源、置信度、限制项  
5. 文档和日志同步更新，保证可交接

---

## 8. 面试可讲的亮点

1. 把 LLM 应用做成有路由、有工具编排的 Agent 系统  
2. 解决了真实产品中的稳定性问题（外部接口波动、兜底策略）  
3. 引入 RAG 与结构化重写，提升可读性与证据性  
4. 用评测集量化优化效果，形成“工程 + 产品”双证据链

## 9. 当前限制与下一步

### 限制
- 部分资料仍为样例数据，真实覆盖度可继续扩展
- 中文编码与历史字符串质量仍需持续清理
- 对复杂追问的上下文融合还可进一步算法化

### 下一步建议
1. 扩充高质量真实资料并增强分块检索策略  
2. 统一文本规范与术语词典，降低表述波动  
3. 强化对比场景上下文补全与一致性测试  
4. 建立线上监控指标（失败率、回退率、耗时）
