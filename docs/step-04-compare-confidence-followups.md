# Step 4 说明文档：对比结论可信度与追问建议

## 1. 目标

把双公司对比从“有结论”升级为“有边界感、有可信度表达、有下一步动作”，增强产品可解释性与实用性。

## 2. 本步改动

- `src/agent.py`
  - 对比结果增加：
    - `confidence_score`（0-100）
    - `industry_context`
    - `limitations`
    - `followups`
- `app.py`
  - 对比卡片展示可信度和行业语境
  - 新增“对比限制说明”
  - 新增“建议继续追问”

## 3. 设计逻辑

- 可信度初始分为 82，按数据完整性扣分：
  - 样例兜底：-12
  - 报告期不一致：-8
  - 行情数据缺失：-5
- 分数范围约束在 40-95，避免虚高或过低误导。
- 限制条件与追问建议面向普通用户表达，不使用复杂模型术语。

## 4. 验收标准

- 对比问题页面可见：
  - 结论可信度分数
  - 行业语境说明
  - 限制条件提示
  - 后续追问建议
- 单公司流程保持不变。

## 5. 测试指引

- 输入：`宁德时代和比亚迪谁更好`
- 预期：
  - 出现“结论可信度：xx/100”
  - 出现“行业语境”
  - 出现“对比限制说明”（如存在）
  - 出现“建议继续追问”列表
