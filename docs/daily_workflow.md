# 每日开发日志流程

## 目的

每天记录完成事项、待办事项、问题和下一步，避免项目推进时丢上下文。

## 日志位置

- 目录：`dev_logs/`
- 命名：`YYYY-MM-DD.md`
- 示例：`dev_logs/2026-05-17.md`

## 使用方式

生成或补齐当天日志：

```powershell
python scripts/dev_log.py
```

生成时附带完成事项和待办事项：

```powershell
python scripts/dev_log.py --done "完成项目文档骨架" --todo "验证 start.bat 启动"
```

## Codex 工作要求

- 每次完成开发任务后，必须更新当天日志。
- 日志要写清楚“完成事项”和“待办事项”。
- 如果发现风险或阻塞，也要记录。
- 不把临时缓存、虚拟环境或密钥写入日志。

## 自动提醒

已配置 Codex 每日提醒：每天 18:00 检查当天开发日志是否更新，并协助补齐完成事项、待办事项、风险与下一步。
