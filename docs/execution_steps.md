# 分步开发执行计划

## 开发节奏

每次只推进一个小目标，完成后更新当天开发日志，再进入下一步。

## Step 1：项目治理和文档

- 建立 `docs/` 文档目录。
- 建立 `dev_logs/` 开发日志目录。
- 建立 `codex.md` 工作指引。
- 建立日志生成脚本。

验收标准：新成员打开 `codex.md` 即可知道项目文件、规范和下一步怎么做。

## Step 2：基础运行稳定性

- 确认 `start.bat` 可以创建虚拟环境并安装依赖。
- 补充 `.env` 配置说明。
- 验证缺少 API Key 时应用可运行。

验收标准：Windows 用户可以双击启动，失败时能看到明确提示。

## Step 3：数据层增强

- 梳理 AKShare 实际返回字段。
- 固化指标提取逻辑。
- 扩展公司别名表。
- 增强数据状态提示。

验收标准：至少 5 家样例公司可以稳定返回结果。

## Step 4：AI 输出质量

- 优化 system prompt。
- 强化 JSON 输出约束。
- 增加合规过滤和兜底话术。
- 手动测试典型问题。

验收标准：输出通俗、结构稳定、不包含直接交易建议。

## Step 5：UI 和演示体验

- 优化布局、状态提示、图表样式。
- 增加加载态和错误态。
- 检查移动窗口和常见桌面分辨率显示。

验收标准：演示时路径清晰，用户能快速理解结论和风险。

## Step 6：验收和整理

- 按 README 执行一次完整启动。
- 跑一次无 API Key 兜底场景。
- 跑一次真实 API 场景。
- 更新开发日志和待办。

验收标准：项目可以交给用户本地运行和演示。
