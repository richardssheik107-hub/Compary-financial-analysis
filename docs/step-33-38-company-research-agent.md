# Step 33-38 公司经营研究 Agent 升级说明

## 目标

本阶段把“大白话财报 Agent”从单纯解释财务摘要，升级为“公司整体经营状况研究 Agent”的第一版基线。目标不是直接训练模型，而是先建立可信资料入口、研究上下文、结构化研究报告、证据来源和质量评测闭环。

## 本次完成内容

### Step 33：资料目录与研究上下文

- 新增 `data/reports/`，作为本地合法年报、季报、研报摘要的资料入口。
- 新增 `src/research_models.py`，定义：
  - `SourceNote`：记录结论依据的来源类型、标题、说明和文件路径。
  - `ResearchDocument`：统一承载本地资料文本。
  - `CompanyResearchContext`：统一承载公司画像、财务指标、业务信息、管理层讨论、研报观点、风险和来源。
  - `ResearchResult`：统一承载最终研究报告。
- 设计原因：先统一数据结构，后续无论接 PDF、网页资料还是向量检索，都不需要重写 UI 和主流程。

### Step 34：财报/研报摘要器

- 新增 `src/report_loader.py`，当前支持 `.txt` 和 `.md` 文件。
- 新增 `summarize_company_sources(company, documents)`，按固定维度抽取：
  - 主营业务
  - 增长驱动/管理层观点
  - 研报观点
  - 风险因素
- 设计原因：先用轻量关键词抽取建立可跑链路，避免一开始引入复杂 RAG 后难以定位问题。

### Step 35：单公司深度研究 Agent

- 新增 `src/research_agent.py`。
- 新增 `run_company_research_agent(query)`，输出单公司经营研究报告。
- 新增 `general_business_qa`，当问题不绑定具体公司时，只输出通用经营问题回答，不强行展示公司图表和研究过程。
- 单公司报告固定包含：
  - 一句话结论
  - 业务概览
  - 经营表现
  - 盈利质量
  - 现金流质量
  - 成长与行业
  - 主要风险
  - 后续跟踪清单
  - 来源依据
  - 可信度
- 设计原因：让输出从“几句大白话”变成可复盘、可面试解释的公司研究框架。

### Step 36：双公司深度对比

- 在研究 Agent 中新增双公司深度对比输出。
- 修复“贵州茅台和宁德时代谁的经营质量更好”这类句式未进入对比分支的问题。
- 对比报告固定包含：
  - 总体结论
  - 业务模式对比
  - 财务质量对比
  - 现金流质量对比
  - 成长性对比
  - 风险对比
  - 后续跟踪清单
- 设计原因：公司对比不能只给一个分数，需要解释为什么更稳、为什么成长性更强、还缺什么资料。

### Step 37：证据引用与可信度

- 每份研究报告包含 `source_notes`。
- 当前来源类型包括：
  - 财务数据
  - 本地资料
  - 年报文本
  - 研报观点
  - 资料边界
- 新增 `confidence_score`，按财务数据完整度、本地资料命中、管理层讨论、研报观点和风险信息加权。
- 没有本地资料支撑的判断会在限制说明中提示。

### Step 38：评测与作品包装

- 新增 `eval/research_testset.csv`，先放入 10 条研究质量样例。
- 新增 `scripts/research_eval_runner.py`，检查：
  - 路由是否正确
  - 研究结构是否完整
  - 是否有来源依据
  - 是否规避买卖建议和目标价
- 设计原因：输出质量不能只靠主观感觉，需要逐步建立可量化的评测基线。

## UI 变化

- 保持原有页面结构不大改。
- 单公司和双公司分析完成后，新增“深度研究报告”展示区。
- 通用问题仍保持简洁回答，不展示图表和研究过程。

## 验收标准

- 应用可以正常启动。
- 输入单公司问题，可以看到原有分析和新增深度研究报告。
- 输入双公司对比问题，可以看到原有对比视图和新增深度对比报告。
- 报告中能看到可信度、后续跟踪清单和依据来源。
- 没有本地年报/研报时，仍能基于财务数据输出，并提示资料不足。

## 测试指引

建议测试：

1. `贵州茅台整体经营状况怎么样`
2. `宁德时代的增长质量如何`
3. `贵州茅台和宁德时代谁的经营质量更好`
4. `平安银行和招商银行哪个更稳`
5. `怎么判断一家公司现金流质量好不好`

预期结果：

- 公司问题应展示深度研究报告。
- 双公司问题应展示深度对比报告。
- 通用问题只展示问题本身的简洁回答。
- 页面不应出现买入、卖出、目标价等交易建议。

## 风险与下一步

- 当前本地资料仅支持 `.txt` 和 `.md`，PDF 解析还未接入。
- 当前摘要器是关键词抽取，后续应升级为“资料切片 + 检索 + LLM 摘要”。
- 当前研究报告主要是规则框架加财务数据，后续应补充真实年报、季报和研报摘要样本。
- 下一步建议优先做 PDF 解析和资料命中可视化，让报告更接近真实公司研究。
