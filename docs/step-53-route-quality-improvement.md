# Step 53：路由质量优化与指标提升

## 目标

在文本质量评测体系建立后，先修复最影响用户体验的路由问题：双公司对比问题没有稳定进入 `compare_companies_deep`。

## 问题定位

Step 51-52 的 50 条离线评测结果显示：

| 指标 | 基线 |
|---|---:|
| route_success_rate | 74.00% |
| text_quality_pass_rate | 100.00% |
| avg_total_score | 84.8 |

失败样本主要集中在 R024-R035：用户输入两个公司进行对比，但系统只识别到其中一个公司，最终降级为单公司研究。

2. 对比意图关键词补充
- 文件：`src/agent.py`
- 补充正常中文关键词：
  - `对比`
  - `比较`
  - `谁更好`

3. 边界问题保护
- 文件：`src/research_agent.py`
- 对 `000001和谁比更合适` 这类“只有一个公司，但询问比较对象”的问题，进入通用问答而不是误判为单公司研究。

## 结果

运行命令：

```powershell
python scripts/text_quality_eval_runner.py --offline --limit 50 --label route-fixed-offline-50
```

| 指标 | 优化前 | 优化后 | 变化 |
|---|---:|---:|---:|
| route_success_rate | 74.00% | 100.00% | +26.00pp |
| text_quality_pass_rate | 100.00% | 100.00% | 0.00pp |
| avg_total_score | 84.8 | 84.9 | +0.1 |
| avg_source_count | 1.8 | 2.3 | +0.5 |

## 决策说明

这一步优先做路由质量，而不是先继续改写作模板。原因是：用户看到的文本质量首先取决于系统是否进入正确分析流程。双公司问题如果被降级为单公司研究，即使文字本身通顺，也不算真正回答问题。
