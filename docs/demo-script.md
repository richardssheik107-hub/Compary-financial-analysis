# Demo 演示脚本（2-3 分钟）

## 场景 1：单公司分析

输入：  
`贵州茅台现在经营质量怎么样，未来一年看什么`

展示重点：
1. 公司识别与分析结果输出
2. 大白话解读与未来看法
3. 评分卡 + 雷达图 + K线图 + 核心财务表
4. Agent 执行流程

讲解话术：  
“这个场景展示单公司链路：先识别目标，再抓取财务数据，然后生成文本解读，最后以图表和文字一起呈现结果。”

---

## 场景 2：双公司对比

输入：  
`600519和300750谁更好`

展示重点：
1. 对比结论卡片（领先方 + 原因）
2. 双公司 K 线图（上下布局）
3. 财务指标对比表
4. 对比文字分析 + 未来看法

讲解话术：  
“这个场景展示对比链路：先抽取两个目标，分别分析，再做综合对比，最后把结论、图表和文字点评统一输出。”

---

## 场景 3：通用问题模式

输入：  
`利润增长但现金流下降说明什么`

展示重点：
1. 未识别具体公司时的通用问答输出
2. 延申问题建议

讲解话术：  
“当用户不指定公司时，系统不会失败，而是切换到通用模式，给出可用的财报理解框架。”

---

## 收尾建议

1. 强调稳定性设计：  
- 失败兜底  
- 上下文记忆  
- 路由分流

2. 强调量化结果：  
- 对比成功率 100%  
- 路由成功率 85%（当前基线）
